# Roadmap: P2P Agent Mesh

> Последнее обновление: 2026-05-10
> Статус: v0.2.0-alpha опубликован, следующие шаги

## Что мы сделали (v0.2.0)

**Гипотеза:** можно построить децентрализованный pub/sub транспорт для AI-агентов без центрального сервера, Redis/Kafka.

**Результат:** гипотеза подтверждена — 32 теста, живая демка, 3 агента обмениваются подписанными Ed25519 сообщениями через IPFS PubSub.

## Ключевой вывод

**IPFS PubSub (CLI) — не подходит для production.** Он решил задачу прототипирования, но:
- 200-500 MB RAM на daemon
- ~20 msg/s пропускная способность
- Зависимость от внешнего бинарника
- Daemon crash = агент молчит

## Что дальше (v0.3)

Следующий шаг — переписать транспорт с нуля. Без IPFS.

Цель: лёгкий P2P транспорт (5-10 MB, 1000+ msg/s), zero-dependency бинарник.

### Архитектура v0.3

```
┌─────────────────────────────────────────────┐
│  sdk/agent.py — AgentMesh API (не меняется) │
├─────────────────────────────────────────────┤
│  core/transport.py — новый P2P транспорт    │
│  • mDNS для локального discovery            │
│  • TCP/QUIC для передачи сообщений          │
│  • at-least-once delivery                   │
│  • автоматический reconnect                │
├─────────────────────────────────────────────┤
│  core/wal.py — SQLite WAL (уже есть, ОК)    │
│  core/identity.py — Ed25519 (уже есть, ОК)  │
│  core/dht.py — Kademlia DHT (уже есть, ОК)  │
│  core/sig_gate.py — rate limit (уже есть, ОК)│
└─────────────────────────────────────────────┘
```

Ключевой принцип: **API агента не меняется.** `AgentMesh(agent_id, [capabilities])` остаётся тем же. Меняется только `transport.py` под капотом.

### Язык реализации

- Транспорт: **Python** (asyncio) или **Rust** (pyo3)
- Решение: Python для быстрого прототипа, Rust для production
- Первая итерация: Python asyncio (TCP sockets)
- Вторая итерация: Rust pyo3 (если есть потребность в скорости)

### Масштабирование

| Этап | Архитектура | Кол-во агентов |
|------|-------------|----------------|
| v0.2 | IPFS CLI обёртка | 3-10 |
| v0.3 | Свой TCP transport | 10-100 |
| v0.4 | Rust ядро + QUIC | 100-1000+ |

## Риски

| Риск | Вероятность | Что делаем |
|------|-------------|------------|
| Свой транспорт сложнее чем кажется | Высокая | Начинаем с TCP, без QUIC в первой итерации |
| Потеря совместимости с IPFS | Средняя | v0.3 не обязана быть обратно совместимой |
| Просадка по времени | Высокая | Закладываем 2-3 недели на transport.py |
| mDNS не работает через Docker | Средняя | Fallback: ручной bootstrap списка peers |

## Как запустить v0.2.0 (сейчас)

```bash
git clone https://github.com/konantgit-sys/p2p-agent-mesh
cd p2p-agent-mesh
pip install cryptography
ipfs daemon --enable-pubsub-experiment &
python examples/3_agent_langgraph.py
```

## Как запустить v0.3 (план)

```bash
pip install p2p-agent-mesh
python -c "from sdk.agent import AgentMesh; a = AgentMesh('node1', ['ping']); await a.start()"
# Никаких внешних зависимостей. Никакого IPFS.
```

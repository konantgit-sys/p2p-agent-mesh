## ФАЗА 0.5 — DHT KADEMLIA (критический пробел)
**Срок: 3-5 дней**
**Зависимости: нет (чистый Python)**

Что сделать:
1. Реализовать Kademlia DHT (K=3):
   - `put(key, value, ttl)` — сохранить
   - `get(key)` → value — найти
   - Репликация на K ближайших узлов
   - Node lookup по XOR-дистанции
2. Заменить `phase0/dht.py` (сейчас in-memory dict) на реализацию
3. Интегрировать с relay: relay = seed-нода
4. Тесты: DHT lookup < 2 сек, репликация работает

**Почему ПЕРВЫМ:** без DHT нет децентрализации — это фундамент

**ЧТО НЕ ДЕЛАТЬ:** не оптимизировать, не делать persistence, не плодить фичи

## ФАЗА 0.6 — RELAY REDUNDANCY
**Срок: 1 день**
**Зависимости: Фаза 0.5 (DHT)**

Что сделать:
1. Запустить второй relay (relay2.v2.site, другой порт)
2. Агенты при старте: DHT lookup → получают список relay seed-нод
3. Если relay1 недоступен → fallback на relay2
4. Тест: отключаем relay1 → агенты продолжают работу через relay2

**Зачем:** устранить SPOF. spec требует 0 SPOF.

## ФАЗА 1.1 — REPUTATION SCORING
**Срок: 2 дня**
**Зависимости: Фаза 0.5 (DHT для хранения репутации)**

Что сделать:
1. `agent.reputation()` — возвращает Score (0.0 - 1.0)
2. Репутация = f(успешные взаимодействия, подписи, age)
3. `query_agents(capability, min_reputation=0.5)` — фильтр
4. Репутация хранится в DHT (не в relay!)

**Почему:** spec требует discovery с репутацией. Без неё discovery неполный.
